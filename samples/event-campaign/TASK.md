# TASK.md

Build a standalone sample website for this folder.

## Sample Identity
- Title: Event Campaign Website
- Slug: `event-campaign`
- Type: Event promotion and registration website

## Objective
- Create a bold event campaign page that communicates date, concept, and registration action clearly.
- The page should convert interest into sign-up or inquiry.
- The final website copy should be written in Japanese for prospective Japanese clients and participants.

## Inputs
- Read `../../AGENTS.md`
- Read `../../SAMPLE_SPEC.md`
- Read `../../CODING_RULES.md`
- Read `../../REVIEW_CHECKLIST.md`
- Read `./COLOR_SCHEMA.md`

## Output
- Create a single-page static campaign website centered on `index.html`
- Keep the registration journey fast and concentrated on one page
- Create `css/styles.css`
- Create `js/scripts.js` only if interaction adds clear value

## Audience
- Japanese users considering attendance at an event or campaign
- Mobile users checking time, location, and registration details
- Visitors motivated by urgency and schedule clarity

## Design Direction
- Mood: festive, urgent, energetic
- Keep the structure punchy and action-oriented
- The page should feel eventful without becoming chaotic

## Content Direction
- Use realistic Japanese copy
- Avoid placeholder text
- Focus on event excitement, key details, and registration confidence
- Keep the tone dynamic but clear

## Required Sections
- Hero:
  Japanese event headline, date or period, registration CTA
- Event Overview:
  Concept, theme, and who it is for
- Program or Highlights:
  Sessions, attractions, or featured content
- Schedule and Access:
  Time, venue, map-style information
- Ticket or Entry Guide:
  Registration details, pricing, or participation flow
- Trust and FAQ:
  Organizer info, support, or common questions
- Final CTA:
  Registration push section
- Footer:
  Contact-style details and navigation

## Interaction Guidance
- Countdown, anchor navigation, or reveal motion is acceptable
- Respect `prefers-reduced-motion`
- Interaction should reinforce urgency and usability

## Visual Constraints
- Follow `COLOR_SCHEMA.md`
- Use accent color strongly for CTA and schedule emphasis
- Keep the page legible even with a high-energy palette

## Success Criteria
- The page looks portfolio-ready
- It reads as a believable Japanese event campaign website
- It works on mobile and desktop
- It can be reviewed using `../../REVIEW_CHECKLIST.md` without obvious gaps
- The Japanese copy feels appropriate for a real event site
